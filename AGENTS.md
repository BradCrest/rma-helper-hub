# AGENTS.md — AutoRMA 工作協議

此 repo 正在執行「無人工RMA計劃」(Zero-Human-Touch RMA Plan)。所有 AI 工具（Codex、Claude Code、Cursor 等）在這個 repo 工作時，**必須遵守以下協議**。

## 計劃檔案

`RMA_ZERO_HUMAN_PLAN.md`（repo 根目錄）是唯一的真相來源，記錄了九個自動化 Phase 的狀態、工作項目、測試記錄與決策點。

| 項目 | 值 |
|------|-----|
| 計劃檔案 | `RMA_ZERO_HUMAN_PLAN.md` |
| 系統入口 | https://rma-helper-hub.lovable.app/admin |
| 說明文件 | https://bradcrest.github.io/rma-helper-hub/ |

---

## 開始工作前（必做）

1. 讀取 `RMA_ZERO_HUMAN_PLAN.md`
2. 找出目前焦點 Phase：
   - 優先選「🔄 進行中」的 Phase
   - 若無，選第一個「📋 待開始」的 Phase
3. 確認該 Phase 的未完成工作項目（`- [ ]`）與任何 `⚠️ 待確認` 項目
4. 向使用者報告：目前焦點、待做項目、需人工確認的問題

---

## 完成工作後（必做）

更新 `RMA_ZERO_HUMAN_PLAN.md`，然後 commit + push：

1. 勾選完成的 checkbox：`- [ ]` → `- [x]`
2. 更新 Phase 狀態（同時更新頂部總覽表格）：
   - 開始第一項：`📋 待開始` → `🔄 進行中`
   - 全部完成，開始驗收：`🔄 進行中` → `🧪 測試中`
   - 驗收通過：`🧪 測試中` → `✅ 完成`
3. 新增測試記錄（若有執行測試）
4. 有決策點時，加 `⚠️ 待確認：[問題]` 並暫停
5. 更新底部「📝 變更記錄」表格

### Commit 格式
```
plan: update Phase X - [簡短說明]
```

---

## 狀態說明

| 符號 | 意義 |
|------|------|
| 📋 待開始 | 尚未啟動 |
| 🔄 進行中 | 開發中 |
| 🧪 測試中 | 功能完成，驗收中 |
| ✅ 完成 | 驗收通過，上線 |
| ⏸️ 暫緩 | 有依賴問題 |
| ❌ 問題 | 測試失敗，需修正 |

---

## 重要原則

- **不假設**：不確定的系統細節先查 codebase，不要猜
- **決策點不跳過**：遇到需人工判斷的地方，標記並停下
- **每次都更新**：完成任何工作後必須更新計劃檔案並 push
- **計劃優先**：若計劃與 codebase 現狀衝突，以計劃為準並請使用者確認
