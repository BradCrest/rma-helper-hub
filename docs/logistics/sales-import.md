# 銷貨匯入

**位置**：物流作業 → 銷貨匯入 Tab

此 Tab 用於將整新品銷售記錄匯入系統，與 RMA 工單或整新品庫存進行關聯。

## 什麼時候需要操作這個 Tab？

- 整新品賣出後，需將銷售記錄同步至系統
- 從 Shopify 或其他銷售管道取得訂單，更新庫存狀態

## 使用情境

CREST 系統販售的整新品來源包括：
- RMA 維修後的整新品（A/B/C 等級）
- 直接由業務出售的庫存機

銷貨匯入確保：
1. 整新品庫存數量準確
2. 銷售記錄與 RMA 來源關聯（可追溯）
3. 財務對帳有記錄依據

## 匯入流程

1. 前往銷貨匯入 Tab
2. 選擇匯入來源：
   - 手動輸入
   - 從 Shopify 同步（若已連接）
3. 確認產品型號、等級、售價、買家資訊
4. 提交後系統更新整新品庫存狀態為「已售出」

## 與整新品庫存的關係

銷貨完成後，對應的整新品庫存記錄狀態更新：

| 庫存狀態 | 說明 |
|---------|------|
| `available` | 可販售（尚未售出）|
| `reserved` | 已預留（客戶訂購中）|
| `sold` | 已售出（銷貨匯入後）|

## 注意事項

- 整新品庫存的主要管理在「供應商維修 Tab → 整新品庫存」子頁
- 銷貨匯入只記錄已完成的銷售，預訂或詢價不在此記錄
- 若需退貨，需聯繫主管處理庫存退回流程
