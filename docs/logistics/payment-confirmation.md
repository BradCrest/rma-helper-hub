# 付款確認

**位置**：物流作業 → 付款確認 Tab

此 Tab 用於確認過保（保固外）維修費用已由客戶繳清，才能繼續進行維修作業。

![付款確認 Tab](/screenshots/logistics-payment.jpg)

## 什麼時候需要操作這個 Tab？

- 客戶已同意維修費用，並表示已完成付款
- 財務確認款項入帳後，在此 Tab 標記確認

## 列表說明

每列顯示：
- RMA 號碼
- 客戶名稱
- 應收金額
- 客戶確認付款日期
- 付款狀態

## 付款確認流程

1. 確認財務收到款項（對帳 / 匯款紀錄）
2. 在 Tab 中找到對應 RMA
3. 點擊「確認付款」
4. 填寫確認備註（如「已收匯款 NT$ 2,000，2025/05/10」）
5. 確認後工單進入 `repairing`（維修中）

## 整新品購買的付款

若客戶選擇購買整新品替換：

1. 確認整新品等級與定價
2. 客戶付款後在此確認
3. 系統自動更新整新品庫存狀態
4. 安排出貨

## 付款方式

目前支援的付款方式（請確認公司最新規定）：
- ATM 轉帳（客戶轉至指定帳戶後提供轉帳後五碼）
- 線上刷卡（透過公司商店頁面）

> 現金收款不建議使用，務必留有書面記錄。

## 注意事項

- 不可在款項確認入帳前進行維修，避免爭議
- 付款確認後才可通知廠商開始維修
- 退款流程（客戶取消）需另行向主管申請，不在此 Tab 處理
