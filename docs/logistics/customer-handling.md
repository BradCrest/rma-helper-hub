# 客戶處理

**位置**：物流作業 → 客戶處理 Tab

此 Tab 讓管理員記錄與客戶的非 Email 聯繫（如電話、LINE 訊息），以及追蹤客戶處理進度。

## 功能說明

### 聯繫記錄列表

顯示所有有聯繫記錄的 RMA 工單，依最新聯繫時間排序。

每列顯示：
- RMA 號碼
- 客戶名稱
- 最近聯繫方式（電話 / Email / LINE / 其他）
- 最近聯繫日期
- 記錄摘要（前 50 字）

### 新增聯繫記錄

在指定 RMA 工單頁面，點擊「新增聯繫記錄」：

| 欄位 | 說明 |
|------|------|
| 聯繫日期 | 實際發生日期（預設今天）|
| 聯繫方式 | 電話 / Email（非 RMA 系統）/ LINE / 其他 |
| 摘要 | 此次聯繫的內容重點 |

> **建議**：每次電話或 LINE 對話後都記錄摘要，方便後續管理員接手時了解狀況。

### 查看歷史記錄

所有記錄依時間排列，可完整追蹤此客戶的聯繫歷程。

## 使用情境

### 客戶主動來電詢問進度
1. 確認客戶 RMA 號碼
2. 在對應工單記錄此次電話內容
3. 若需後續行動，在備註中標記（如「待確認維修費用」）

### 需要主動聯繫客戶
1. 在 Dashboard 或 RMA 列表找到需聯繫的工單
2. 電話聯繫客戶
3. 回到系統記錄聯繫結果

### 轉手給其他管理員
記錄的歷史可讓接手的管理員立即了解：
- 客戶關心的問題點
- 已承諾給客戶的事項
- 尚未解決的問題

## 與 Email 回覆的差異

| 項目 | 客戶處理（手動記錄）| RMA 回覆（Email）|
|------|----------------|----------------|
| 聯繫方式 | 電話 / LINE / 面對面 | Email |
| 記錄方式 | 管理員手動輸入摘要 | 系統自動記錄 |
| 客戶端可見 | 否 | 是（Email 寄給客戶）|
| 正式性 | 內部記錄 | 正式書面往來 |

## 注意事項

- 聯繫記錄為**內部記錄**，客戶看不到
- 若在電話中承諾了特定事項（如維修期限、費用），請務必記錄，以免後續糾紛
- 若客戶表達強烈不滿，建議在備註中標記「需升級處理」並通知主管
