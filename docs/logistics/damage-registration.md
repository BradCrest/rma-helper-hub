# 故障登記

**位置**：物流作業 → 故障登記 Tab

此 Tab 用於產品到貨後，登記實際故障狀況及初步診斷結果，為後續維修方案的決策依據。

![故障登記 Tab](/screenshots/logistics-fault.jpg)

## 什麼時候需要操作這個 Tab？

當 RMA 工單已確認收件（`received`），開始進行產品檢測時。

## 列表說明

顯示所有已收件但尚未完成故障登記的 RMA 工單。

每列顯示：
- RMA 號碼
- 客戶名稱
- 產品型號 / 序號
- 收件日期
- 保固狀態（保固內 / 保固外 / 無法判定）
- 客戶描述的故障問題

## 故障登記流程

1. 找到目標 RMA，點擊「登記故障」
2. 填寫以下資訊：
   - **實際故障狀況**：技術人員的初步診斷描述
   - **外觀等級**（整新品分級用）：A / B / C
   - **維修方案**：建議的維修方式
   - **預估費用**（過保時必填）
3. 儲存後系統自動更新工單進入後續流程

## 故障記錄後的流向

```
故障登記完成
  ↓
  ├── 保固內 → 直接安排出貨（更新為 `paid`，進入出貨處理 Tab）
  │
  └── 保固外 → 進入「待客戶確認」（`contacting`）等候費用同意
              └── 若客戶不同意維修 → 更新為 `no_repair`，退回產品
```

## 整新品外觀分級

在故障登記時同步評估外觀等級（供整新品庫存使用）：

| 等級 | 外觀狀況 | 功能 |
|------|---------|------|
| A | 近新品，無明顯磨損 | 完全正常 |
| B | 有輕微磨損，日常使用常見 | 完全正常 |
| C | 有明顯刮傷 / 掉漆 | 完全正常 |

> 若功能有問題（如螢幕亮點、電池容量明顯衰退），不應評為整新品，應退回工廠重修或列為報廢。

## 注意事項

- 故障登記是後續流程的關鍵依據，描述需具體（如「螢幕左側有橫向亮線 2 條」）
- 保固外的費用估算需事先查詢維修報價，確認後才能通知客戶
- 若序號無法解碼，此時可手動覆蓋保固判斷（見保固審核 Tab）
