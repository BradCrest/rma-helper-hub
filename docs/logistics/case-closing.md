# 結案追蹤

**位置**：物流作業 → 結案追蹤 Tab

此 Tab 追蹤已完成出貨的工單，確認客戶確實收到產品後進行正式結案。

![結案追蹤 Tab](/screenshots/logistics-closing.jpg)

## 什麼時候需要操作這個 Tab？

- 出貨後，確認包裹已送達客戶
- 需要追蹤物流狀態，確認無異常

## 列表說明

顯示已出貨（outbound 物流已填寫）的工單，追蹤交付狀態。

每列顯示：
- RMA 號碼
- 客戶名稱
- 出貨日期
- 快遞商 / 追蹤號
- 預估送達日
- 客戶確認收件狀態

## 結案流程

此 Tab 顯示所有 `shipped_back*`（已寄回）及 `follow_up`（後續追蹤）狀態的工單。

```
出貨完成（shipped_back_* 或 no_repair → shipped_back_original）
  ↓ 追蹤物流（3-7 天），確認客戶收件
  ↓
  ├─ 設定後續關懷 → 更新為 follow_up（設定關懷到期日）
  │     ↓ 關懷完成
  │   更新為 closed
  └─ 直接結案 → 更新為 closed
```

## 追蹤物流

可直接點擊追蹤號碼連結查看物流狀態，或複製追蹤號到快遞商網站查詢。

## 異常處理

### 客戶反映未收到
1. 先查詢物流追蹤狀態
2. 若顯示「已投遞」但客戶說沒收到：
   - 確認地址是否正確
   - 請客戶查詢鄰居 / 門房是否代收
   - 聯繫快遞商確認簽收記錄
3. 若確認包裹遺失：向快遞商提出理賠申請

### 包裹退回（無法投遞）
若包裹因地址錯誤或無人簽收被退回：
1. 確認退件原因
2. 聯繫客戶重新確認地址
3. 重新寄件，填寫新的追蹤號
4. 在備註說明「重寄」原因

## 結案後

正式結案後，工單進入「客戶關懷 Tab」，設定維修後追蹤排程。
