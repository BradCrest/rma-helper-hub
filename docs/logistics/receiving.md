# 收件處理

**位置**：物流作業 → 收件處理 Tab

此 Tab 讓管理員確認客戶寄回的產品，並記錄收件資訊。

![收件處理 Tab](/screenshots/logistics-receive.jpg)

## 什麼時候需要操作這個 Tab？

當客戶填寫寄件資訊（RMA 狀態變為 `shipped`），且包裹實際送達倉庫時。

## 列表說明

顯示所有 `shipped`（已寄出）、`received`（已收到）、`inspecting`（檢測中）狀態的 RMA。

每列顯示：
- RMA 號碼
- 客戶名稱
- 產品型號 / 序號
- 客戶填寫的快遞商 / 追蹤號
- 客戶預計寄出日期
- 等待天數（自寄出日起算）

### 逾期警示

若等待超過 **14 天**仍未確認，該列會以橘色 / 紅色標示，提醒管理員跟催。

可點擊「逾期未到」按鈕篩選這些工單。

## 確認收件流程

1. 找到目標 RMA，點擊「確認收到」
2. 填寫以下資訊：
   - **實際到貨日期**（預設今天）
   - **外觀狀況備註**（選填）：如外箱破損、配件齊全等
3. 確認後，系統自動：
   - 更新 RMA 狀態為 `received`
   - 寫入 inbound 物流記錄（`rma_shipping`）
   - 寫入狀態變更歷史（`rma_status_history`）

## 收件後的下一步

確認收件後，需評估維修方案：

### 保固內 → 免費維修
1. 檢查產品問題
2. 若需送廠 → 前往「供應商送修追蹤」加入批次
3. 自行維修完成 → 前往「出貨處理 Tab」填寫出貨資訊，更新為 `shipped_back_original`

### 保固外 → 客戶自費
1. 評估維修成本
2. 透過「RMA 回覆」通知客戶診斷結果與費用
3. 等待客戶確認後再進行維修

### 使用整新品替換（建議選項）
若倉庫有同型號的整新品（A/B/C 等級），可建議客戶以整新品替換：
1. 前往供應商送修追蹤 → 整新品庫存 Tab 查看庫存
2. 確認適合的等級與報價
3. 透過「RMA 回覆」傳送診斷通知 Email（含整新品報價）

> **診斷通知 Email**：系統提供固定模板，依保固狀態（保固內/外）自動帶入不同說明文字。

## 寄件資訊不一致的處理

若客戶填寫的追蹤號對不上：
1. 查詢快遞商官網確認包裹動態
2. 若包裹已延誤，透過「RMA 回覆」聯繫客戶
3. 若包裹已遺失，協助客戶向快遞商索賠（CREST 不負責運送過程中的損失）

## Outbound 寄件記錄

維修完成後，管理員將產品寄回客戶，需在工單詳情頁記錄：
- 快遞商（如黑貓、順豐）
- 追蹤號碼
- 寄出日期

記錄後進入「出貨處理 Tab」填寫寄回資訊，更新為 `shipped_back_*`（依產品情況選擇）。
