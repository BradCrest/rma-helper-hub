# 物流作業總覽

物流作業頁面（`/admin/logistics`）是管理員日常操作最頻繁的區域，以 **Tab 分頁**組織六個功能模組。

## Tab 分頁一覽

| Tab 名稱 | 主要用途 | 主要對象 |
|---------|---------|---------|
| [RMA 回覆](/logistics/rma-reply) | Email thread 對話管理 | 客戶（RMA 專屬信件）|
| [客戶來信](/logistics/customer-email) | Gmail 收件匣處理 | 客戶（一般來信）|
| [收件管理](/logistics/receiving) | 確認產品到貨 | 倉庫/收件端 |
| [客戶處理](/logistics/customer-handling) | 聯繫記錄追蹤 | 客服人員 |
| [供應商送修追蹤](/logistics/supplier-repair) | 送廠維修進度 | 技術/採購 |
| [後續關懷追蹤](/logistics/followup) | 維修後追蹤關懷 | 客服人員 |

## 典型工作流程

### 新 RMA 進來時

```
客戶申請
  → 1. RMA 回覆 Tab：確認/回覆客戶詢問
  → 2. 等待客戶寄件（shipped 狀態）
  → 3. 收件管理 Tab：確認到貨並記錄
  → 4. 評估維修方案
     ├── 自行維修 → 更新狀態為 repairing → completed
     └── 送廠 → 供應商送修追蹤 Tab
  → 5. 完成後 → 後續關懷追蹤 Tab 設定追蹤日期
```

### 每日例行工作建議順序

1. **客戶來信 Tab** — 處理 Gmail 收件匣新信件（避免漏信）
2. **RMA 回覆 Tab** — 查看有未讀回覆的 RMA thread
3. **收件管理 Tab** — 確認新到的包裹
4. **供應商送修追蹤 Tab** — 確認有無工廠完工需驗收
5. **後續關懷追蹤 Tab** — 確認今日到期的關懷排程

## 各 Tab 的篩選器

每個 Tab 都有頂部篩選列，共同特性：
- **關鍵字搜尋**：搜尋 RMA 號碼、客戶名稱、序號
- **狀態篩選**：依各 Tab 的狀態選項篩選
- **逾期標記**：快速顯示超過時限的項目

篩選器為即時篩選，輸入或選擇後立即更新列表（資料已在本地快取）。
