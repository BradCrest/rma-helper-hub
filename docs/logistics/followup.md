# 後續關懷追蹤

**位置**：物流作業 → 後續關懷追蹤 Tab

此 Tab 讓管理員追蹤維修完成後的客戶關懷，包括後續聯繫排程與滿意度調查問卷發送。

## 功能說明

### 關懷列表

顯示所有已設定後續關懷的 RMA 工單，包含：
- RMA 號碼
- 客戶名稱
- 維修完成日
- **預計關懷日期**（帶顏色 Badge）
- 問卷狀態

### 日期 Badge 顏色說明

| 顏色 | 條件 | 說明 |
|------|------|------|
| 🔴 紅色 | 已逾期 | 超過預計關懷日仍未完成 |
| 🟡 黃色 | 3 天內到期 | 即將到期，應儘速處理 |
| 🟢 綠色 | 今日到期 | 今天就是關懷日 |
| ⬜ 灰色 | 未來日期 | 尚未到期 |

### 問卷狀態 Badge

| 狀態 | 說明 |
|------|------|
| 待發送 | 問卷尚未寄出 |
| 已發送 | 問卷已寄出 |
| 已填寫 | 客戶已完成問卷 |

## 建立關懷排程

出貨完成（`shipped_back_*`）後，建議立即設定關懷排程：

1. 在 RMA 工單詳情頁，找到「後續關懷」區塊
2. 點擊「設定關懷日期」
3. 填寫：
   - **預計關懷日期**（建議：維修完成後 7 天）
   - **關懷方式**：Email / 電話
4. 若選擇 Email，可選擇是否附上滿意度問卷
5. 儲存後出現在關懷追蹤列表

## 發送關懷 Email

到了預計關懷日，管理員應：

1. 在關懷列表找到今日到期的工單
2. 點擊「傳送關懷郵件」
3. 選擇是否包含調查問卷（`includeSurvey`）
4. AI 輔助起草內容（選填：點擊「AI 起草」）
5. 審閱或修改草稿
6. 點擊「傳送」

### 包含問卷 vs 不包含問卷

| 選項 | 適用情境 |
|------|---------|
| 不含問卷 | 一般關懷詢問，維修滿意且無爭議 |
| 含問卷 | 希望收集滿意度數據，或此案有特殊情況需要追蹤 |

問卷連結由系統自動產生，客戶點擊後可填寫 1-5 分評分和文字意見。

## 查看問卷回覆

若客戶填寫了問卷：
- 問卷狀態自動更新為「已填寫」
- 在工單詳情頁的「客戶回饋」區塊可查看評分和意見
- 問卷資料存於 `rma_customer_feedback` 表

## 篩選器

| 篩選 | 說明 |
|------|------|
| 搜尋 | RMA 號碼 / 客戶姓名 |
| 問卷狀態 | 待發送 / 已發送 / 已填寫 |
| 逾期未完成 | 超過預計關懷日仍未發送 |

## 關懷追蹤的重要性

後續關懷不只是禮貌，更有實際效益：
- **發現潛在問題**：維修後若有再次故障，客戶可能直接反映
- **收集滿意度數據**：作為服務品質改善依據
- **建立客戶忠誠度**：主動關懷的品牌印象優於等客戶抱怨

> **建議**：每天上班先查看關懷列表，確保沒有逾期未處理的項目。
