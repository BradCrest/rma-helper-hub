# 供應商送修追蹤

**位置**：物流作業 → 供應商送修追蹤 Tab

此功能追蹤將產品送交外部維修供應商（工廠）進行維修的完整流程，以及管理整修後的整新品庫存。

## 供應商說明

CREST 目前合作的維修供應商：

| 供應商代號 | 中文名稱 | 負責型號 |
|-----------|---------|---------|
| `chuangbao` | 創葆 | CR-4、CR-1 |
| `zhengnengliang` | 正能量 | CR-5、CR-5L |

> **注意**：CR-F 型號目前**不送廠**（由 CREST 自行處理）。  
> CR-1 送創葆。CR-5 和 CR-5L 送正能量。  
> 型號 → 供應商的對應由系統自動帶入，管理員可手動覆蓋。

## 概念說明

### 送修工單 vs 批次

```
送修工單（rma_supplier_repairs）
  每台機器一筆，對應一筆 RMA
  
批次（supplier_repair_batches）
  將多台機器打包成一批寄給供應商
  一個批次內可有多台機器（同一供應商）
```

**原因**：供應商無法配合一台一台處理，通常要累積數台後一起出貨。

## 送修追蹤 Tab

### 列表欄位

| 欄位 | 說明 |
|------|------|
| RMA | 對應的 RMA 工單號 |
| 客戶 | 客戶姓名 |
| 型號 / 序號 | 產品資訊 |
| 供應商 | 創葆 / 正能量（Badge）|
| 狀態 | 目前進度（Badge）|
| 送出日 / 在外天數 | 送到工廠的日期，及目前已在外幾天 |
| 預估 / 實際費用 | NT$ 格式，含預估與發票金額 |

### 工單狀態流程

```
[pending_send] 待寄出
     ↓ 加入批次並出貨
[at_factory] 工廠維修中
     ↓ 工廠回報完工
[repaired] 工廠完工
     ↓ 管理員驗收確認
[returned] 已驗收完成
```

特殊狀態：
- `scrapped`（報廢）— 工廠認定無法維修，或驗收後品質不符

### 篩選器

| 篩選 | 說明 |
|------|------|
| 搜尋 | RMA 號碼 / 客戶名稱 / 序號 |
| 供應商 | 全部 / 創葆 / 正能量 |
| 狀態 | 全部 / 各狀態 |
| 逾期未回（>30天）| 在工廠超過 30 天仍未完工的工單 |

> 逾期警示：**在外超過 30 天**的工單，天數欄位以紅色粗體顯示。建議每週檢查一次。

## 批次管理面板

### 建立新批次

1. 點擊「建立送修批次」
2. 選擇供應商（創葆 / 正能量）
3. 輸入批次備註（選填，如「5月第一批」）
4. 系統建立一個草稿批次（`draft` 狀態）

### 將工單加入批次

1. 找到要送修的工單（`pending_send` 狀態）
2. 點擊工單的「加入批次」
3. 選擇目標草稿批次
4. 確認後，工單與批次建立關聯

### 確認出貨

草稿批次備齊所有要出貨的機器後：
1. 進入批次詳情
2. 確認批次內所有工單的清單
3. 點擊「確認出貨」
4. 填寫：
   - 出貨日期
   - 快遞商
   - 追蹤號
5. 確認後，批次狀態改為 `shipped`，所有工單自動改為 `at_factory`

### 批次狀態

| 狀態 | 說明 |
|------|------|
| `draft` | 草稿，尚未出貨，可繼續添加工單 |
| `shipped` | 已寄出，工廠處理中 |
| `received` | 工廠已全部回寄收回 |

## 更新單筆工單

點擊列表的「檢視 / 更新」開啟工單詳情 Dialog：

### 可更新的欄位

| 欄位 | 說明 |
|------|------|
| 供應商狀態 | 更新工單進度 |
| 送到工廠日期 | 實際出廠日（批次出貨時自動填入）|
| 工廠回傳日期 | 收到工廠回寄的日期 |
| 維修費用（預估）| 送廠前的費用估算（NT$）|
| 維修費用（實際）| 工廠發票金額（NT$）|
| 發票單號 | 工廠發票號碼 |
| 等級評定 | A / B / C（整新品品質等級）|
| 維修描述 | 工廠回傳的維修說明 |
| 備註 | 內部備註 |

### 等級評定說明

工廠完工、管理員驗收後，依產品品相分等：

| 等級 | 說明 | 建議處理 |
|------|------|---------|
| A | 完全修復，外觀近新 | 整新品定價最高，優先銷售 |
| B | 功能正常，有輕微磨損 | 標準整新品定價 |
| C | 功能正常，外觀明顯瑕疵 | 折扣整新品定價 |

> **等級由管理員手動決定**（系統不自動判定），在驗收後於工單詳情中選擇。

## 整新品庫存 Tab

切換到「整新品庫存」子分頁，管理所有已驗收整新品的庫存狀態。

### 庫存列表

| 欄位 | 說明 |
|------|------|
| 來源 RMA | 對應的原始 RMA |
| 型號 | 產品型號 |
| 序號 | 產品序號 |
| 等級 | A / B / C |
| 定價 | 系統依型號和等級自動建議報價 |
| 狀態 | 目前庫存狀態 |
| 入庫日期 | 驗收完成日 |

### 整新品狀態

| 狀態 | 說明 |
|------|------|
| `in_stock` | 在庫（可供銷售）|
| `used_warranty` | 已撥用保固（用於替換給其他保固客戶）|
| `sold` | 已售出（整新品銷售）|
| `scrapped` | 報廢（二次檢查後品質不合格）|

### 整新品定價參考

系統自動顯示建議報價（依型號 + 等級）：

| 型號 | A 級 | B 級 | C 級 |
|------|------|------|------|
| CR-4 | 依設定 | 依設定 | 依設定 |
| CR-5L | 依設定 | 依設定 | 依設定 |

> 實際報價可依個案調整，此為參考依據。

### 整新品撥用流程（替換保固客戶）

當有客戶需要整新品替換（保固內）：
1. 在整新品庫存中找合適的機器（型號相符、等級 A 或 B）
2. 在客戶 RMA 工單詳情中記錄替換整新品的序號
3. 更新整新品狀態為 `used_warranty`
4. 更新 RMA 狀態為 `completed`

## 費用追蹤

系統提供兩個費用欄位：

| 欄位 | 填寫時機 | 說明 |
|------|---------|------|
| 預估維修費用 | 送廠前 | 根據故障描述估算，用於成本預算 |
| 實際維修費用 | 工廠完工後 | 對應工廠發票金額 |

差異可用來評估報價準確度。

> **注意**：費用單位為新台幣（NT$）。系統在顯示時會自動加上千分位符號。

## 常見操作情境

### 情境 1：正常維修流程

```
1. 客戶 RMA 收到後 → 在工單建立送修工單（pending_send）
2. 累積到足夠台數 → 建立批次
3. 出貨給工廠 → 批次改為 shipped，工單改為 at_factory
4. 工廠完工通知 → 更新工單狀態為 repaired + 填入實際費用
5. 收回機器驗收 → 更新狀態為 returned + 填入等級
6. 機器入整新品庫存 → 更新整新品狀態
```

### 情境 2：工廠認定無法維修

1. 工廠回報 → 更新工單狀態為 `scrapped`
2. 通知客戶（透過 RMA 回覆）
3. 協商後續處理（退費 / 換機方案）
4. RMA 最終狀態更新為 `cancelled` 或 `completed`（視結果）

### 情境 3：逾期追蹤

1. 每週檢查「逾期未回（>30天）」篩選
2. 聯繫供應商確認進度
3. 在工單備註記錄催貨結果與預計回廠日
