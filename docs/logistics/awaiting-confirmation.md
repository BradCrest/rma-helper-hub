# 待客戶確認

**位置**：物流作業 → 待客戶確認 Tab

此 Tab 管理所有等候客戶回覆確認的 RMA 工單，主要用於過保（保固外）維修的費用確認場景。

![待客戶確認 Tab](/rma-helper-hub/screenshots/logistics-customer.jpg)

## 什麼時候需要操作這個 Tab？

- 工單為保固外維修，已通知客戶維修費用，等候客戶決定
- 管理員傳送報價或方案說明後，進入此狀態追蹤

## 列表說明

每列顯示：
- RMA 號碼
- 客戶名稱
- 產品型號
- 等候天數（自通知日起算）
- 預估維修費用
- 上次聯繫日期

### 逾期提醒

若等候超過 **7 天**仍無回覆，該列會以橘色標示，建議進行追蹤聯繫。

## 標準處理流程

```
保固外診斷完成
  ↓ 透過「RMA 回覆」傳送診斷通知 Email（含報價）
  ↓ 工單進入「待客戶確認」
  ↓
  ├── 客戶同意維修 → 進入「付款確認」或直接 repairing
  ├── 客戶拒絕維修 → 更新為 cancelled，退回產品
  └── 客戶要求整新品替換 → 確認庫存後安排換機
```

## 診斷通知 Email

系統提供標準診斷通知模板，在「RMA 回覆」Tab 傳送：

**Email 內容自動帶入**：
- 保固狀態（保固內 / 保固外）
- 產品型號 / 序號
- 故障描述
- 維修方案建議
- 整新品報價（A/B/C 等級，依型號）

傳送後請在工單備註記錄「已傳送診斷通知，等候客戶確認」。

## 等候逾時的處理

| 等候天數 | 建議行動 |
|---------|---------|
| 7 天無回覆 | 透過 Email 或電話主動追蹤 |
| 14 天無回覆 | 再次傳送提醒，告知將在 7 天後退件 |
| 21 天無回覆 | 視同放棄，更新狀態為 `cancelled`，退回產品 |

> 退件時，在備註說明「客戶無回覆，逾時退件」，並更新狀態為 `cancelled`。

## 常見問題

**Q：客戶說要考慮，但超過一週沒消息？**
先傳送一封提醒 Email（可使用知識庫 AI 起草），告知預計的等候截止日。

**Q：客戶表示費用太高，要求打折？**
費用調整需主管核准。核准後在備註說明，再透過 RMA 回覆更新客戶。
