# 系統設定

**位置**：`/admin/settings`（頂部導覽：系統設定）

設定頁讓管理員管理常用檔案、AI 模型參數，以及管理員帳號（部分功能限 `super_admin`）。

![系統設定上半](/rma-helper-hub/screenshots/settings-top.jpg)

## 常用檔案庫

**位置**：設定頁最上方

上傳保固政策、產品說明書、報價單範本等常用文件，日後在「RMA 回覆」傳送信件時可一鍵插入為附件。

**支援格式**：PDF、文件（DOC/DOCX）、圖片等  
**單檔上限**：25 MB

### 上傳步驟

1. 填寫分類（選填，例如：保固、報價、說明書）
2. 點擊「選擇檔案上傳」
3. 上傳後顯示於列表，含：名稱、分類、大小、使用次數、上傳時間

### 操作按鈕

| 按鈕 | 功能 |
|------|------|
| ⬇ 下載 | 下載檔案到本機 |
| ✏ 編輯 | 修改分類標籤 |
| 🗑 刪除 | 刪除檔案（不可回復）|

> 目前系統中已有 6 個檔案（CR-LITE 說明書、Diverout 說明書等）。

## 管理員管理（super_admin 限定）

### 新增管理員

1. 輸入用戶電子郵件
2. 點擊「新增」
3. **注意**：用戶必須先在管理登入頁面註冊帳號，才能被新增為管理員

### 待審核申請

若有用戶在登入頁申請成為管理員，會顯示在此：
- 點擊「核准」→ 系統在 `user_roles` 建立記錄
- 點擊「拒絕」→ 申請被標記為 `rejected`

目前若無待審核申請，顯示「目前沒有待審核的申請」。

### 管理員列表

顯示所有管理員帳號，包含：
- Email
- 角色（admin / super_admin — 超級管理員）
- 建立時間

**重設密碼**：點擊管理員帳號旁的選項 → 系統寄送密碼重設連結至該 Email。

**移除管理員**：從 `user_roles` 刪除記錄。該帳號仍存在於 Auth，但無法進入後台。

> 只有 `super_admin` 可以移除其他管理員，且無法移除自己。

## 登入記錄

顯示最近的管理員登入記錄，供稽核使用：

| 欄位 | 說明 |
|------|------|
| 電子郵件 | 登入的管理員帳號 |
| 事件類型 | 登入 / 登出 |
| 時間 | 事件時間戳記 |
| IP 地址 | 登入來源 IP（含地理位置）|

**使用情境**：
- 確認新管理員是否已成功登入
- 偵測異常登入（不明 IP / 異常地理位置）
- 資安稽核

## AI 模型設定

> **注意**：AI 設定頁面目前可能只對 `super_admin` 顯示。

| 設定項目 | 說明 | 建議值 |
|---------|------|--------|
| LLM Provider | AI 服務提供商 | Anthropic（Claude）|
| Model | 使用的模型 | `claude-opus-4-5`（品質最佳）/ `claude-sonnet-4-5`（速度平衡）|
| Temperature | 回覆的隨機性（0~1）| 0.3~0.5（保持一致性）|
| Max Tokens | 最大輸出長度 | 1000~2000 |

**Temperature 說明**：
- **0.0~0.2**：回覆非常固定，適合高一致性需求
- **0.3~0.5**：稍有變化，適合客服回覆（推薦）
- **0.7~1.0**：創意性較高，不適合客服場景

修改後點擊「儲存設定」，立即生效（呼叫 `update-ai-settings` Edge Function）。

## Email 佇列設定

| 設定項目 | 說明 |
|---------|------|
| Batch Size | 每次 cron 執行寄幾封（建議 5~20）|
| Send Delay (ms) | 每封之間的延遲毫秒數（建議 100~500）|

> 調整 batch_size 和 send_delay_ms 可控制寄信速率，避免被 Email 服務商限制。
