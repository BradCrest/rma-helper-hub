# 知識庫管理

**位置**：`/admin/email-knowledge`

知識庫是 AI 輔助回覆功能的核心。管理員上傳的文件、FAQ、歷史 Email 都會被向量化（Embedding），供 AI 在起草回覆時語意搜尋參考。

## 知識來源類型

| 類型（source_type）| 說明 | 適合內容 |
|------------------|------|---------|
| `faq` | 常見問題集 | 保固政策、維修流程、費用說明 |
| `template` | 回覆範本 | 標準回覆格式、常用句型 |
| `email` | 歷史 Email | 過往成功的客服回覆案例 |
| `document` | 一般文件 | 產品說明書、維修手冊 |

## 上傳知識文件

### 方式一：文字貼上

1. 點擊「新增知識」
2. 選擇來源類型
3. 輸入標題
4. 貼上或輸入文字內容
5. 儲存

### 方式二：檔案上傳

1. 點擊「上傳文件」
2. 支援格式：`.txt`、`.md`、`.pdf`（純文字）
3. 上傳後系統自動提取文字內容
4. 可在儲存前預覽內容

> **PDF 注意事項**：僅支援以文字為主的 PDF（非掃描版圖片 PDF）。若上傳圖片 PDF，文字提取可能失敗或不完整。

### 方式三：從 Gmail 匯入

1. 點擊「從 Gmail 匯入」
2. 搜尋特定的 Email 對話
3. 選擇要加入知識庫的信件
4. 系統自動以 `source_type = 'email'` 建立知識來源

> 需要 Gmail OAuth 授權才能使用此功能。

## 向量化（Embedding）

新增知識後，系統會非同步進行向量化：

```
新增知識文件
    ↓ upload-knowledge-file / kickoff-email-embedding-job
    ↓ generate-email-embeddings（chunking + embedding）
    ↓ 寫入 email_embeddings（pgvector）
    → 狀態：pending → completed
```

向量化狀態顯示在知識列表的 Badge：
- **pending**：排隊中或處理中
- **completed**：已完成，可被 AI 搜尋
- **failed**：向量化失敗（通常因 OpenAI API 錯誤）

若狀態長時間停在 `pending`，可以點擊「重新向量化」手動觸發。

## 知識搜尋（RAG 流程）

AI 起草回覆時，系統後台執行：

```
管理員問題 / 客戶信件內容
    ↓ 向量化問題
    ↓ pgvector 語意搜尋（top-N 相關知識）
    ↓ 組成 prompt：
      "以下是相關知識：{知識片段}
       請根據以上知識，幫我回覆：{信件內容}"
    ↓ Claude API 生成回覆
    ↓ 顯示草稿
```

## 管理現有知識

### 搜尋

頂部搜尋框可搜尋知識標題和內容關鍵字。

### 篩選

依 `source_type` 篩選（FAQ / 範本 / Email / 文件）。

### 編輯

點擊知識項目，可修改標題和內容。修改後需重新向量化（系統自動觸發）。

### 刪除

刪除知識文件同時刪除對應的 embeddings。刪除後 AI 不再參考此知識。

> **注意**：刪除是不可回復的操作，請確認後再執行。

## 知識庫建置建議

剛上任時，建議優先建立：

1. **保固政策 FAQ**（5-10 條）
   - 如何判斷保固期？
   - 過保費用怎麼計算？
   - CR-4/CR-1/CR-5L 各自的問題類型

2. **標準回覆範本**（5-10 條）
   - 申請確認回覆
   - 請客戶提供購買憑證
   - 維修完成通知（保固內 / 保固外）
   - 整新品報價

3. **常見故障問答**
   - 螢幕問題處理流程
   - 電池問題說明
   - 防水失效處理

知識庫愈豐富，AI 起草的品質愈好。
