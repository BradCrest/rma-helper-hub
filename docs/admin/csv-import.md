# CSV 批次匯入

**位置**：`/admin/csv-import`

提供兩種 CSV 匯入功能：
1. **歷史資料匯入**：從舊系統遷移完整 RMA 記錄（51 欄格式）
2. **批次新申請**：經銷商多筆申請（精簡格式）

## 歷史資料匯入（51 欄格式）

### 適用情境
- 從舊系統（如 Excel、舊資料庫）遷移 RMA 記錄
- 批次補建歷史工單

### CSV 格式要求

- 編碼：**UTF-8**（必須）
- **第一行**：說明文字（備忘用，系統忽略）
- **第二行**：欄位標題（繁體中文欄位名稱）
- **第三行起**：實際資料
- 分隔符：英文逗號（`,`）
- 報修單號和客戶姓名為必填欄位
- 文字欄位若含逗號，需用雙引號包覆（`"內容,含逗號"`）

### 51 欄位對應（重要欄位）

| 欄位名稱（CSV）| 欄位說明 | 必填 |
|--------------|---------|------|
| 工單號 | RMA 號碼（若為空，系統自動產生）| 否 |
| 客戶姓名 | | ✅ |
| 客戶Email | | ✅ |
| 客戶電話 | | ✅ |
| 產品型號 | CR-4 / CR-1 / CR-5L 等 | ✅ |
| 序號 | 產品序號 | 否 |
| 購買日期 | YYYY-MM-DD 格式 | 否 |
| 故障類型 | 螢幕問題 / 電池問題 / 其他 | ✅ |
| 故障描述 | 詳細說明 | ✅ |
| 狀態 | 工單目前狀態（中文）| ✅ |
| 建立時間 | YYYY-MM-DD HH:MM:SS | 否 |

> 完整的 51 欄位清單，請參考系統 `csvParser.ts` 中的 `CSV_COLUMN_MAP`。

### 狀態對應（中文 → 系統 enum）

| CSV 中文狀態 | 系統狀態 |
|------------|---------|
| 待處理 | `pending` |
| 處理中 | `processing` |
| 已寄出 | `shipped` |
| 已收到 | `received` |
| 維修中 | `repairing` |
| 已完成 | `completed` |
| 已取消 | `cancelled` |

### 匯入步驟

1. 前往 `/admin/csv-import`
2. 點擊「選擇 CSV 檔案」或拖拉檔案到上傳區
3. 系統自動解析並顯示預覽（前 10 列）
4. 確認欄位對應無誤
5. 點擊「開始匯入」
6. 系統逐列匯入，完成後顯示：
   - 成功匯入數
   - 失敗數（含錯誤原因）
   - 跳過數（重複 RMA 號碼）

### 常見錯誤

| 錯誤訊息 | 原因 | 解決方式 |
|---------|------|---------|
| 必填欄位為空 | 客戶姓名等欄位空白 | 補齊資料或跳過該列 |
| 狀態值無效 | 狀態文字不在對應表中 | 確認中文狀態完全匹配 |
| 日期格式錯誤 | 非 YYYY-MM-DD 格式 | 轉換日期格式 |
| RMA 號碼重複 | 系統已有相同工單號 | 匯入會自動跳過，不更新 |

## 批次新申請（精簡格式）

### 適用情境
- 經銷商一次送多台機器，使用統一格式申請
- 管理員代填多筆新申請

### 匯入入口

在 RMA 申請表單（`/`）切換到「多筆申請」Tab，上傳 CSV。

也可以在 `/admin/csv-import` 選擇「批次新申請」模式。

### 精簡格式欄位

| 欄位 | 說明 |
|------|------|
| 產品型號 | CR-4 / CR-5L 等 |
| 序號 | |
| 故障描述 | |
| 備註 | 選填 |

> 客戶資訊（姓名、Email 等）填寫一次，套用到所有列。

---

## 匯入注意事項

::: warning 注意
- **匯入是不可回退的**：一旦匯入成功，工單就建立在系統中。若需要刪除，需透過 Supabase Dashboard 手動操作，且刪除會留下 `rma_deletion_logs` 快照。
- **測試環境先試**：正式大量匯入前，建議先用少量資料（5-10 列）測試，確認格式正確。
- **重複檢查**：若 CSV 中的 RMA 號碼已在系統中存在，系統會跳過（不更新）。若要更新，需手動在工單詳情頁修改。
:::

## 匯入後確認

匯入完成後，建議：
1. 前往 `/admin/rma-list`，依建立時間排序，確認新工單已出現
2. 抽查幾筆工單，確認欄位資料正確
3. 若有保固需要自動判斷，確認序號 / 型號填寫正確（系統會自動計算）
